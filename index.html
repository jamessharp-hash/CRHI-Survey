Supabase connection

Paste your Supabase project URL and anon key here. Once saved, responses from both surveys will be stored automatically. These credentials are saved only in your browser and embedded in the page when you deploy.

Step 1 — Create your tables

Go to your Supabase project → SQL Editor → New query. Paste and run this:

create table adult_responses (
  id            uuid default gen_random_uuid() primary key,
  created_at    timestamptz default now(),
  school        text not null,
  visit_date    text, zip_code text,
  concern_type  text, first_visit text,
  q1_satisfaction   int,
  q2_resolved       text,  q3_followup_care  text,
  q4_recommend      text,  q5_community      text,
  q6_quality        text,  q7_informed       text,
  q8_followup_comm  text,  q9_return_work    text,
  q10_avoided_absence text, q11_distance     text,
  q12_time_saved    text,  q13_would_have    text[],
  q14_provider      text,  q15_payment       text[],
  q16_referral      text[], comments         text
);

create table parent_responses (
  id            uuid default gen_random_uuid() primary key,
  created_at    timestamptz default now(),
  school        text not null,
  visit_date    text, zip_code text, grade_level text,
  concern_type  text, first_visit text,
  q1_satisfaction   int,
  q2_resolved       text,  q3_followup_care  text,
  q4_recommend      text,  q5_benefitted     text,
  q6_quality        text,  q7_informed       text,
  q8_followup_comm  text,  q9_return_class   text,
  q10_avoided_absence text, q11_stayed_work  text,
  q12_distance      text,  q13_time_saved    text,
  q14_would_have    text[], q15_provider     text,
  q16_payment       text[], q17_referral     text[],
  comments          text
);

Step 2 — Set access policy (Row Level Security)

Run this in the SQL editor to allow the form to insert responses but not read or delete them:

alter table adult_responses enable row level security;
alter table parent_responses enable row level security;

create policy "Allow anonymous inserts" on adult_responses
  for insert to anon with check (true);

create policy "Allow anonymous inserts" on parent_responses
  for insert to anon with check (true);

Step 3 — Deploy to GitHub Pages

1. Save your credentials above
2. In your GitHub repo, edit index.html
3. Find the two lines near the bottom that read SUPABASE_URL and SUPABASE_KEY and paste in your values
4. Commit — your live site updates in about 60 seconds
